Detailed Description

template<class RequestType, class AnswerType, class QIDType>
class Smart::IQueryServerPattern< RequestType, AnswerType, QIDType >

The server part of the Query pattern to perform two-way (request-response) communication.

Template parameters

  • RequestType: request class (Communication Object)
  • AnswerType: answer (reply) class (Communication Object)
  • QIDType: the QueryId that should be implemented for each middleware by subclassing IQueryId

status code:
  • SMART_OK : everything is ok and answer sent to requesting client
  • SMART_WRONGID : no pending query with that id known
  • SMART_DISCONNECTED : answer not needed anymore since client got disconnected meanwhile
  • SMART_ERROR_COMMUNICATION : communication problems
  • SMART_ERROR : something went wrong
